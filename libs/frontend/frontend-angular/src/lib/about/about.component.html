Beschrijving van project

Motard.com is een website waarop gebruikers reviews kunnen achterlaten op motoren. Gebruikers kunnen motoren browsen, gegevens van de motoren bekijken, en reviews bekijken van andere gebruikers en zelf hun eigen reviews achterlaten. Reviews worden geplaats op modellen, niet op unieke exemplaren van eigenaren.

Users kunnen ook gangs aanmaken en aan gangs deelnemen. Een 'Gang' is een groep motorrijders die bijelkaar horen (e.g. Hell's Angels). Een gang is een groep motorrijders die dezelfde interesses hebben. Bij de detail pagina van een gang worden de motoren getoond die de leden van de gang leuk vinden.
Hiermee kan de user zien of de gang motoren leuk vinden die hij/zij zelf ook leuk vind, en kan daarna besluiten om deel te nemen aan de gang.

Op elke motor review pagina krijgt de gebruiker te zien welke motoren andere users ook leuk vinden naast de huidige motor. Hiermee krijgt de gebruiker aanbevelingen voor andere motors te zien die hij misschien, net als de huidige motor, interessant zou vinden.
Naast motor die geliked zijn door users die de huidige motor leuk vinden, worden er ook motoren getoond die users leuk vinden ten opzichte van de huidige motor. Hiermee kan de user zien welke motoren andere users leuk vonden tegenover de huidige motor die zij NIET leuk vinden.
Ook laat de pagina zien bij welke gangs er veel leden zijn die deze motor leuk vinden, wat nog meer helpt met het zoeken van gangs die de user leuk zou vinden.

Uiteraard kunnen users registeren en inloggen, en hebben ze user rechten op basis van hun rol. Er is een "User" en "Admin" rol Het aanmaken van motoren mag alleen gedaan worden door gebruikers met de "Admin" rol. Wanneer een motor is toegevoegd kunnen alle users de nieuwe motor bekijken en reviews achterlaten. Users kunnen alleen gegevens wijzigen en verwijderen van entiteiten die zij zelf hebben gecreerd, OF als de huidige gebruiker de "Admin" rol heeft. Een admin kan bij alle gegevens van alle entiteiten en alle CRUD functionaliteiten uitvoeren, ongegacht of hij de eigenaar is van die entiteiten.

Non-Functional Requirements

ID Non-Functional Requirement Acceptatiecriteria Prioriteit
0.1 De webapplicatie moet authorisatie toepassen op basis van de rol van de ingelogde gebruiker.
  • Een user kan alleen zijn eigen entiteiten (eigen gangs en reviews en user gegevens) wijzigen of verwijderen.
  • Alleen een admin mag motoren aanmaken/wijzigen/verwijderen
  • Een admin heeft toegang tot bijna alle CRUD functionaliteiten binnen de webapplicatie, inclusief die van andere users.
 Must
0.2 Bij alle forms moet er correcte validatie toegepast worden.
  • Een user kan alleen entiteiten aanmaken/wijzigen indien de formulier kloppen is zonder validatie fouten.
  • De validatie bij alle velden van formulieren moet logisch en passend zijn
  • Wanneer er validatie fouten opkomen, moet dit duidelijk worden aangegeven aan de gebruiker zodat de user op de hoogte is van de velden die moeten worden aangepast.
 Must
0.3 De gebruiker moet op de hoogte blijven van de status van de web applicatie.
  • Wanneer er elementen worden geladen van de backend, moet de webapplicatie dit duidelijk aangeven
  • Wanneer er een fout op treedt moet de webapplicatie dit duidelijk tonen aan de gebruiker.
